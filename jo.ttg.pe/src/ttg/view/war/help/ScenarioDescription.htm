Scenario Description Keys

This group of keys set descriptive values applicable to the entire scenario.

Name This keys set the name of the scenario. It should only appear once.

Author It is reccomended that this the name and e-mail address of the person who wrote this scenario in the form "name" . This is not enforced. It should only appear once.

Version Since you may end up modifying a scenario multiple times, this allows you to track a version number with it. Is not enforced. It should only appear once.

Description This lets you give a prose introduction to the scenario. Explain the design concepts, the player goals, what fun you had making it, or whatever you want. It can appear multiple times. Subsequent appearances add to the current description (with a space inbetween).
