Side Definition Keys

These keys are used to define the values for each side. They all start with 'Side' and a number for that side, followed by the rest of that key. Below we will use Side1, but it is equally applicable for Side2, etc.

Side1Name This is the name of the side. It should only appear once.

Side1VP This is a starting value to a side's Victory Points. It can be used to round off values so each side starts with the same. It should only appear once.

Side1Resources This is the amount of initial resources this side starts with. Resources are used in ship building. It should only appear once.

Side1WorldURI This key declares territory that belongs to this side at the start of the game. This key is in TtgUriFormat. You can use a world or ord URI to specify a single world You can use a sector or subsector URI to specify a whole area. This key can be repeated. Each addition adds more territory.

Side1Ship This key adds a ship to that side. The value for this key is in the format: 

        Side1Ship = 'name','att','def','cap','jump'
        Side1Ship = 'name','att','def','cap','jump',x'num'

Where 'name" is the name of the ship, 'att' is the attack factors, 'def' is the defense factors, and 'jump' is the jump range in parsecs (hexes).

'cap' can be of the form '#.#' or '#' or '.#'. The first number says how many factors of capacity the ship can contain. For example, specifying a capacity of '2' would create a hanger large enough for a '1.1.0' fighter. The second number says how many extra jump-1 amounts of fuel should be stored. For example, specifying a capacity of '.1' would create an extra fuel tank with enough space to jump this ship an extra 1 parsec.

'num' is an optional prameter to indicate the number of these ships to add to this side.

Note: in actual play there is no distinction made between hanger capacity and fuel capacity. Nothing prevents you putting fuel into your hanger space or ships into your fuel space (although not both at once!). This is given as a convienence here in ship design.

This key can be repeated. Each addition adds another ship.
